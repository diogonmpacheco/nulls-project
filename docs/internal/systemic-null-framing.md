# Systemic Null Framing

The Nulls Project should make one idea explicit:

Some nulls may deserve to be considered much more seriously than current consumer genetics or pharmacogenomics labels suggest.

A pharmacogenomic report often translates a gene into a medication rule. That is useful, but it can hide the larger biological state. A true inherited null can mean the body lacks an enzyme or function wherever that gene is normally active. That is different from:

- a clinical poor-metabolizer bucket;
- partial residual function;
- an inhibitor that suppresses only selected compartments;
- a medication-only interpretation.

## The Core Question

The project should ask:

> Is this just a drug-response label, or is this a systemic functional deficiency?

For some genes, the answer may remain medication-specific. For others, especially when expression and function exist outside the liver, the null state may need to be treated more like a body-wide missing-function condition.

## CYP2D6 As The Flagship

`CYP2D6` is the first strong example because it combines:

- inherited no-function alleles;
- whole-gene deletion;
- copy-number and hybrid-allele complexity;
- medication-induced functional nulls;
- liver/systemic pharmacokinetics;
- source-linked human brain expression and function evidence;
- exposure and accumulation cases such as solanidine.

This does not mean every CYP2D6 null phenotype is known. It means the old frame is too small. The question is not only "which drugs need review?" The deeper question is "what does a body miss when CYP2D6 function is absent?"

## Disease-Like Seriousness Without Overdiagnosis

The project can say:

> A true null may be a systemic enzyme-function deficiency that deserves serious study.

The project should not say:

> This variant is automatically a disease diagnosis.

That distinction matters. A formal disease category requires evidence, phenotype mapping, clinical consensus, and safety review. But absence of a formal diagnosis should not force the project to describe a body-wide missing enzyme as merely a pharmacogenomic curiosity.

## Evidence Rule

Every systemic-null claim should be broken into compartments:

- liver / systemic circulation;
- brain / CNS;
- barrier tissues;
- gut;
- heart or target tissues;
- kidney / urine;
- adipose or storage compartments;
- endocrine / reproductive tissues;
- immune or inflammatory hypotheses;
- unresolved peripheral tissues.

For each compartment, the atlas should label:

- expression evidence;
- functional evidence;
- inherited-null expectation;
- medically induced-null expectation;
- accumulation or exposure concern;
- uncertainty.

This keeps the framing serious without becoming speculative.

## Product Rule

The product should make systemic nulls visible early.

Users should be able to see when a record is:

- mostly a medication-response null;
- a candidate systemic deficiency;
- a protective null;
- an unresolved null observation;
- an induced functional null.

This is the main reason The Nulls Project should remain separate from Diognosis. Diognosis can answer medication review questions. The Nulls Project can preserve the larger biological map.
